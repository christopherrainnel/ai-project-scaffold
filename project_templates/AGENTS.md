# Agent Rules (Free Starter)

> Universal entry point for AI agents in this generated project.

## Start Here

1. Start from the active tool entry surface for the tool or IDE.
2. Read `AGENTS.md`.
3. Read `docs/FILE_MAP.md`.
4. Read `ops/AI_WORKFLOW.md`.
5. Follow startup and resume checks from `ops/AI_WORKFLOW.md`.
6. Read only task-relevant docs and files after that.

## Read Triggers

- Architecture, feature, release, or UX work -> also read `docs/ARCHITECTURE.md`, `docs/DECISIONS.md`, and `docs/USER_CONSUMER_JOURNEY_CHECKLIST.md`.
- Privacy, legal, or policy wording work -> also read `docs/PRIVACY.md` and `docs/TERMS.md`.
- Local app or test work -> use repo-local environments and `python -m ...` commands when applicable.

## Rules

- Plan before code.
- Never read or modify `.env`; update `.env.example` only.
- Never expose secrets, private URLs, keys, or tokens.
- Never run destructive commands without explicit approval.
- Treat governance files as protected.

## Closeout

- Run applicable checks from `ops/QUALITY_GATES.md`.
- Update `CHANGELOG_AI.md` after every completed task.
- Update `docs/DECISIONS.md`, `docs/PRIVACY.md`, `docs/TERMS.md`, `docs/THREAT_MODEL.md`, `ops/RUNBOOK.md`, and `docs/USER_CONSUMER_JOURNEY_CHECKLIST.md` when affected.
