# AI Change Log

> Every AI-made change is recorded here. Keep one date block per day and add the newest same-day entry at the top of that block.

## Entry Format

```md
## YYYY-MM-DD

### HH:MM - Short task title
**Files**: list of files changed
**Verification**: how the change was verified
**Notes/Risks**: anything to watch out for
```

## Optional Fields

- `Commands`: include only when command details are materially useful for reproduction, environment recovery, incident handling, or another non-obvious operational step.

## Ordering Rules

- Newest date block first.
- Within a date block, newest timestamp first.
- Leave older dates unchanged unless a dedicated cleanup task says otherwise.
